PreviewType.Readable



Description

Preview will be made available in RAM and will be accessible to the CPU.

Since NatCam v1.2, the preview's pixel buffer is stored natively, and not in the managed layer (Unity). As a result, calls like Get/SetPixel(s) will fail on the preview texture. To have access to the pixel data on the CPU, you are advised to use native code (C, C++, or Objective C) and the OnNativePreviewUpdate callback.

If need access to the pixel data in Unity, call NatCam.PreviewFrame (which does exactly what NatCam did pre- version 1.2). Make sure to comment '#define ALLOCATE_NEW_FRAME_TEXTURES' at the top of NatCamNativeInterface.cs to make sure a new Texture2D doesn't get allocated each time you call NatCam.PreviewFrame.
